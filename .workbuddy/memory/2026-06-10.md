## 打印追溯单性能优化

### 问题
点击"打印追溯单"后连接打印机时程序卡死，必须强制关闭。

### 根因
`PrintDialog.PrintDocument()` 是同步阻塞调用，在与打印机通信时会冻结 UI 线程。

### 修复
1. **异步打印**：用 `XpsDocumentWriter.WriteAsync()` 替代同步 `PrintDocument()`
   - `PrintQueue.CreateXpsDocumentWriter(printQueue)` 静态方法获取写入器
   - `WriteAsync(paginator, jobName)` 异步写入，通过 `WritingCompleted` 事件等待结果
2. **30秒超时自动取消**：`CancellationTokenSource(30s)` + `writer.CancelAsync()`
3. **打印进度浮层**：HistoryView.xaml 添加半透明遮罩+旋转动画+状态文字
4. **新增属性**：`IsPrinting`/`IsNotPrinting`/`PrintStatus`（ObservableProperty）
5. **按钮禁用**：打印中按钮自动禁用（IAsyncRelayCommand 内置 IsRunning）
6. **MotorReportWindow 同步修复**：报告窗口的"打印报告"按钮同样改为异步

### 关键代码位置
- `ViewModels/HistoryViewModel.cs` — `PrintTraceAsync()` 异步方法 + 打印状态属性
- `Views/MotorReportWindow.xaml.cs` — `PrintButton_Click` 改为 async void
- `Views/HistoryView.xaml` — 打印进度浮层（Panel.ZIndex=1000）
